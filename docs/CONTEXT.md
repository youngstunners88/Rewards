# Docs Workspace

User-facing documentation. Written for teachers, not developers.

## What lives here

- `/api/` — Reserved for future. The current app has no API.
- `/guides/` — How-to guides. "How to add a student", "How to start a new class", etc.
- `/changelog/` — One file per release. Format: `YYYY-MM-DD.md`.

## Audience

The primary reader is a teacher. They are not technical. They want to know how to do one thing, not understand the system.

## Voice

- Plain language. No jargon.
- Short sentences.
- Numbered steps for procedures.
- Screenshots welcome (add them to a folder next to the guide).

## Guide template

```md
# How to [do the thing]

What this guide does, in one sentence.

## Steps
1. First step.
2. Second step.
3. Third step.

## If something goes wrong
- If X happens, try Y.
```

## Changelog template

```md
# YYYY-MM-DD

## What's new
- ...

## What's fixed
- ...

## How to use it
- One sentence.
```

## What good docs look like

- A teacher can scan it in 30 seconds.
- Each guide answers exactly one question.
- The changelog tells teachers what changed without making them read code.
